Jethro Pastoral Ministry Manager v1.2 - Readme

Installation Instructions

There are just five steps to get Jethro up and running:

  1. Copy all files to a web-accessible directory on your web host
  2. Create a mysql database for your system to use
  3. Copy the appropriate sample configuration file for your country (eg conf.php.sample.au) to conf.php
  4. Edit conf.php and fill in the essential details (system name, URL, database details). Further explanation is given inside the file.
  5. Point your web browser to the system's URL; the installer will start automatically and will prompt you for details to create the initial staff member account. When the installer is finished it will prompt you to log in to the installed system.

Once the system is running it's a good idea to delete this readme.html file from the server, so as to avoid providing system details to any would-be intruders.

How Jethro stores data

The following is a quick summary of what things Jethro stores, and how they relate to each other

  • Each person belongs to exactly one family. A family is a group of one or more persons who live at the same address and are in some sense related to each other.
  • Address and home telephone number are stored per family (since a family must live at the same address). Email, mobile and work telephone numbers are stored per person
  • Persons have a status field indicating their relationship to the church. Their status may be "archived" in which case they are not normally shown in lists.

  • Families have a status field indicating their currency in the system. Their status may be "archived" in which case they are not normally shown in lists.

  • Each person belongs to exactly one congregation, unless their status is "contact" in which case they can be congregation-less
  • Each person is a member of zero or more groups
  • Each person has zero or more person notes
  • Each family has zero or more family notes
  • A family note or person note can be assigned to a staff member for action
  • A jethro user is a person who can log into the system and have notes assigned to them
  • A system administrator is a user with full permissions, who can create and edit user accounts and congregations
  • Attendance (present, absent or blank) can be recorded for any person for a particular date.
  • A person report is a stored set of parameters which can be executed to display a list of persons filtered and displayed according to various criteria

Why Jethro PMM is called Jethro PMM

Jethro is designed to facilitate efficient team ministry, so it's named after Moses' father-in-law, who in Exodus 18:13-23 introduced Moses to the concept of delegation.
